5. Scaling & Parallelism

July 22 – August 1 — Prof. Li

Overview

The final and longest module ties everything together. Students learn how real AI systems scale beyond a single process — using multiprocessing, multithreading, and distributed training — and then apply these ideas to a self-chosen capstone project, producing reproducible code and a final presentation/demo.

Learning Objectives

  • Differentiate multiprocessing from multithreading and know when to use each in Python
  • Parallelize a CPU-bound task using Python's multiprocessing module
  • Explain the basics of distributed and data-parallel training
  • Simulate MPI-style communication patterns (broadcast, scatter, gather, reduce)
  • Apply scaling techniques to a self-chosen project
  • Produce reproducible code and deliver a short presentation/demo

Phases

Phase 1 — Multiprocessing & Multithreading

The building blocks of parallel programs.

  • Threads vs. processes, and why Python's Global Interpreter Lock (GIL) matters
  • Use multiprocessing to parallelize a CPU-bound task (e.g., processing a large batch of images)
  • Use threading for an I/O-bound task and compare speedups
  • Benchmark single-process vs. multi-process runtimes

Phase 2 — Distributed Training & Data Parallelism

How training scales across multiple workers and devices.

  • Concepts: data parallelism vs. model parallelism, gradient synchronization, and batch splitting
  • Simulate data-parallel training by splitting a dataset across multiple worker processes
  • Compare training time and results to a single-process baseline

Phase 3 — MPI-Style Workflows

The communication patterns behind large-scale computing.

  • Core MPI patterns: send/receive, broadcast, scatter, gather, and reduce
  • Simulate these patterns with mpi4py on a toy distributed computation
  • Connect MPI patterns back to how distributed training synchronizes gradients

Phase 4 — Capstone Project

Apply everything you've learned this summer to a project of your choice.

  • Option A: Image classifier at scale — parallelize data loading and training for a larger dataset
  • Option B: Recommendation system — parallelize similarity computation or matrix factorization across users/items
  • Option C: Parallelized data analysis — process a large dataset using multiprocessing and compare to a serial baseline
  • Deliverable: a reproducible code repository (clear setup instructions, scripts, and results)

Phase 5 — Presentations & Demos

Show what you built.

  • Short presentation (slides or live demo) covering the project's goal, approach, and results
  • Discuss what was parallelized, the speedup achieved, and any tradeoffs encountered
  • Peer Q&A and feedback
