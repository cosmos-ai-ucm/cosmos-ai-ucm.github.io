4. Efficient AI & Edge Deployment

July 20–21 — Prof. Yu

Overview

A model that performs well in a notebook isn't always a model that can run on a phone, a sensor, or a robot. This module looks at the gap between "it works" and "it runs efficiently" — covering model compression techniques and what it takes to deploy a model to resource-constrained edge hardware.

Learning Objectives

  • Explain why model size, latency, and power consumption matter for real-world AI
  • Describe quantization and pruning as model compression techniques
  • Apply post-training quantization to a trained model and measure the size/accuracy/latency tradeoff
  • Export a model to a deployment-ready format (e.g., ONNX or TFLite)
  • Benchmark a model under simulated edge constraints and discuss design tradeoffs

Day-by-Day Activities

Day 1 (Jul 20) — Model Compression

Make models smaller and faster without starting over.

  • Take a model trained earlier in the program and measure its size, latency, and accuracy as a baseline
  • Apply post-training quantization (e.g., float32 → int8) and re-measure
  • Explore pruning: remove low-importance weights/channels and observe the accuracy/size tradeoff
  • Build a table comparing baseline vs. compressed models

Day 2 (Jul 21) — Edge Deployment

Get a model ready for the real world.

  • Export a trained model to ONNX or TFLite format
  • Run inference with the exported model and compare outputs to the original
  • Benchmark inference time under simulated edge constraints (e.g., limited CPU threads/memory)
  • Discussion: how would you choose a model for a self-driving car's onboard computer vs. a cloud server?
